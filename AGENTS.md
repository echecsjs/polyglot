## Project Overview

Read-only reader for Polyglot opening book `.bin` files. Accepts raw bytes and a
Zobrist hash, returns matching book moves via binary search. No runtime
dependencies — keep it that way.

Named exports: `lookup()`. Types: `Entry` (extends `Move` from
`@echecs/position`).

## Architecture Notes

- **ESM-only** — the package ships only ESM. Do not add a CJS build.
- No runtime dependencies — keep it that way. `@echecs/position` is a type-only
  import (compile-time only).
- Single file: `src/index.ts` contains everything — `Entry` type, move decoding,
  binary search, and the `lookup` function.
- Binary search operates directly on the raw `Uint8Array` using `DataView` for
  big-endian reads. No intermediate parsing step.
- Castling moves are stored in Polyglot format as king-to-rook (e.g. `e1h1`) and
  translated to standard king-destination (e.g. `e1g1`) on decode.
- The Polyglot "learn" field (4 bytes per entry) is ignored — unused in
  practice.
- Test fixture at `src/__tests__/fixtures/test.bin` is generated by
  `build-fixture.ts`. Re-run the builder if the fixture needs updating.
